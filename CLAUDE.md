## Project Overview

Tinyscript is a proof-of-concept JavaScript-like scripting language built on Eclipse Xtext. It supports dynamic typing, prototypal inheritance, closures, and sandboxed execution. The project targets multiple deployment targets: Eclipse IDE plugin, standalone REPL, Spring Boot web app, and JVM ScriptEngine.

## Build Commands

This is a Maven multi-module project using Eclipse Tycho for OSGi/Eclipse plugin builds.

```bash
# Build everything (runs from repo root)
mvn clean install

# Build only the REPL fat JAR
mvn clean package -pl de.mkbauer.tinyscript.repl

# Build only the web demo fat JAR
mvn clean package -pl de.mkbauer.tinyscript.webdemo

# Skip tests during build
mvn clean install -DskipTests
```

## Running Tests

### After any code change ŌĆö always run the standalone tests first

After changing anything in the interpreter (`TinyscriptEngine`) or the generator (`TinyscriptGenerator`), always run:

```bash
mvn test -pl de.mkbauer.tinyscript.standalone.tests
```

This is the fastest feedback loop (< 5 s, no OSGi startup) and catches both interpreter regressions and transpiler/interpreter divergence. If node.js is not on PATH the differential tests skip cleanly; the interpreter reference tests always run.

### Test modules

**`de.mkbauer.tinyscript.standalone.tests`** ŌĆö plain Maven/JUnit module, no OSGi:

Tests run in this order (alphabetical class order enforced by Surefire):

1. `InterpreterReferenceTest` ŌĆö 29 `assertEquals(expected, interpreterOutput(tsCode))` tests; one per language feature; failure shows the exact wrong value produced by the interpreter.
2. `TranspilerTest` ŌĆö differential tests; `assertSameOutput(tsCode)` runs each snippet through both the interpreter and `node`, diffs stdout; generated JS is shown on failure so it can be pasted into `node` directly.
3. `TranspilerVerifyTest` ŌĆö integration tests; `assertTranspilerCorrectFromFile(filename)` runs complete `.ts` programs through both interpreter (must not throw) and node (must exit 0).

Base class `TinyscriptTranspilerTestBase` provides:
- `interpreterOutput(String tsCode)` ŌĆö runs snippet through a fresh `TinyscriptEngine`, returns captured stdout; use with JUnit's `assertEquals` for targeted interpreter tests
- `assertSameOutput(String tsCode)` ŌĆö Level 1 differential assertion
- `assertTranspilerCorrect(String)` / `assertTranspilerCorrectFromFile(String)` ŌĆö Level 2 integration assertions
- `assertInterpreterThrows(Class, String)` ŌĆö expected-exception helper

```bash
# Run standalone tests only (fast, use after interpreter or generator changes)
mvn test -pl de.mkbauer.tinyscript.standalone.tests

# Run a single test class
mvn test -pl de.mkbauer.tinyscript.standalone.tests -Dtest=InterpreterReferenceTest

# Run a single test method
mvn test -pl de.mkbauer.tinyscript.standalone.tests -Dtest=InterpreterReferenceTest#testRecursiveFibonacci
```

**`de.mkbauer.tinyscript.tests`** ŌĆö OSGi/Eclipse plugin tests via `tycho-surefire-plugin`:

Tests the interpreter and language features via Xtext's `XtextRunner` / `@InjectWith`. Slower (requires OSGi container startup). Run after confirming the standalone tests pass.

```bash
# Run all OSGi tests
mvn test -pl de.mkbauer.tinyscript.tests

# Run a specific test class
mvn test -pl de.mkbauer.tinyscript.tests -Dtest=ExpressionsTest

# Run a specific test method
mvn test -pl de.mkbauer.tinyscript.tests -Dtest=ExpressionsTest#testIntAddition
```

Test helper base class `TinyscriptInterpreterTestHelper` provides:
- `evaluateSimpleExpression(String expr)` ŌĆö evaluates a single expression, returns `TSValue`
- `executeScriptFromString(String script)` ŌĆö executes inline script
- `executeScriptFromFile(String filename)` ŌĆö loads `.ts` file from `resources/`
- `engine` ŌĆö direct access to `TinyscriptEngine`

mvn clean test

## Architecture

### Module Layout

| Module | Purpose |
|---|---|
| `de.mkbauer.tinyscript` | Core: grammar, interpreter, runtime, validation, scoping |
| `de.mkbauer.tinyscript.tests` | OSGi/Eclipse plugin tests (XtextRunner) |
| `de.mkbauer.tinyscript.standalone.tests` | Standalone (no-OSGi) tests: interpreter reference + transpiler differential |
| `de.mkbauer.tinyscript.ide` | Language server / IDE support (generated by Xtext) |
| `de.mkbauer.tinyscript.ui` | Eclipse editor UI plugin |
| `de.mkbauer.tinyscript.repl` | Standalone REPL app |
| `de.mkbauer.tinyscript.webdemo` | Spring Boot web sandbox demo |
| `de.mkbauer.tinyscript.target` | Eclipse target platform definition |

### Execution Pipeline

```
Source text
  ŌåÆ Xtext parser (generated from Tinyscript.xtext)
  ŌåÆ AST (EMF model objects)
  ŌåÆ TinyscriptValidator (semantic checks)
  ŌåÆ TinyscriptEngine (tree-walking interpreter)
  ŌåÆ TSValue result
```

### Key Classes in `de.mkbauer.tinyscript`

**Interpreter:**
- `interpreter/TinyscriptEngine.java` ŌĆö main tree-walking evaluator; handles all AST node types via visitor-style `switch`/`instanceof` dispatch
- `interpreter/ExecutionContext.java` ŌĆö per-call-frame variable scope; forms a chain for closure lookup
- `interpreter/GlobalExecutionContext.java` ŌĆö global scope; initializes built-in objects at startup
- `interpreter/TSValue.java` ŌĆö wraps all runtime values (number, string, boolean, null, undefined, object, function); handles type coercion
- `interpreter/TSObject.java` ŌĆö property map + prototype reference; backing store for all objects and functions
- `interpreter/ResourceMonitor.java` ŌĆö sandboxing: enforces limits on wall time, CPU time, statement count, call depth, and memory

**Runtime built-ins** (`runtime/` subdirectories): `array/`, `string/`, `math/`, `object/`, `function/`, `fs/`, `system/`

**Grammar:** `Tinyscript.xtext` ŌĆö Xtext grammar defining the full language syntax. After editing, regenerate with `GenerateTinyscript.mwe2`.

### Language Differences from JavaScript

- Semicolons are optional (newline-terminated)
- Variables are block-scoped (no `var` hoisting)
- `for (x in array)` iterates values, not keys
- Arrow functions use `=>` syntax
- No `this` binding differences ŌĆö follows prototypal model

### Dependency Injection

The core module uses Google Guice (wired by Xtext). Bindings are configured in `TinyscriptRuntimeModule`. Tests use `@Inject` via `@RunWith(XtextRunner.class)` + `@InjectWith(TinyscriptInjectorProvider.class)`.

## Running the Applications

```bash
# REPL (after building fat JAR)
java -jar de.mkbauer.tinyscript.repl/target/tinyscript-repl-*.jar

# Web demo (after building fat JAR)
java -jar de.mkbauer.tinyscript.webdemo/target/tinyscript-webdemo-*.jar

# Web demo via Docker
cd de.mkbauer.tinyscript.webdemo
docker build -t tinyscript-webdemo .
docker run -p 8080:8080 tinyscript-webdemo
```
